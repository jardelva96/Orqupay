# Security Policy

## Escopo

Vulnerabilidades de seguranca devem ser reportadas de forma responsavel.

## Reporte

1. Nao abra issue publica com exploit detalhado.
2. Envie relatorio privado para o canal de seguranca do mantenedor.
3. Inclua passos de reproducao e impacto.

## Requisitos minimos

- TLS 1.2+
- assinatura de webhook por HMAC
- mascaramento de dados sensiveis em logs
- rotacao de segredos
